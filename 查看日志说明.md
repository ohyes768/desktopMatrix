# 查看 DesktopWidget 日志的方法

## 📁 日志文件位置
**主日志文件：** 桌面上的 `DesktopWidget_log.txt`

## 🔍 如何查看日志

### 方法1：直接查看日志文件
1. 回到桌面
2. 找到 `DesktopWidget_log.txt` 文件
3. 用记事本或其他文本编辑器打开

### 方法2：Windows命令提示符
```cmd
type %USERPROFILE%\Desktop\DesktopWidget_log.txt
```

### 方法3：PowerShell
```powershell
Get-Content $env:USERPROFILE\Desktop\DesktopWidget_log.txt
```

## 📋 日志内容说明

日志包含以下关键信息：
- `[时间戳] 应用程序启动`
- `[时间戳] 开始添加任务`
- `[时间戳] 任务标题: 'xxx', 选中象限: Q1`
- `[时间戳] 调用 TaskManager.AddTaskSync`
- `[时间戳] 数据库返回ID: xxx`
- `[时间戳] 任务添加成功！`

如果出现错误，会记录：
- `[时间戳] 添加任务时发生异常：xxx`
- `[时间戳] 异常类型: xxx`
- `[时间戳] 堆栈跟踪：xxx`
- `[时间戳] 内部异常：xxx`

## 🧪 测试步骤

1. 运行应用
2. 尝试添加一个任务
3. 如果失败，立即查看日志文件
4. 根据日志内容定位问题

## 🚨 常见错误类型

### 数据库错误
- "无法打开数据库文件" - 检查权限
- "no such table" - 数据库未正确初始化

### 参数错误
- "标题为空" - 输入框内容问题
- "象限无效" - 选择逻辑问题

### 系统错误
- "Access denied" - 权限问题
- "File not found" - 文件路径问题
